PosterReview

An agentic reviewer for research posters.

Upload a poster, get a design score and evidence-based feedback — grounded in what the poster actually measures, and run entirely on local compute at no cost.

Zero API cost Runs locally / on-device Deterministic design metrics Retrieval-grounded

What it does

Peer review is slow; poster feedback is even scarcer — most authors get none until they're standing at the board. PosterReview gives a poster a fast, structured review: a measured design score and specific, research-aware content feedback, so an author can improve the poster before it's printed.

It's modeled on the spirit of Stanford's agentic paper reviewer, but built for the poster medium — which changes both what's possible and what's honest to claim (see the evaluation).

How it works

  1. Parse & measureThe poster PDF is parsed to exact design metrics — font sizes, WCAG contrast, text coverage, column structure, palette — read directly from the file's vector data (or via OCR for flattened posters). These are measurements, not guesses.
  2. Ground in prior workSalient claims become search queries against the free arXiv API; related papers are retrieved to judge novelty and contextualization.
  3. ReviewA local open model reads the poster's text, the measured design facts, and the retrieved related work, and writes a structured review across research-content dimensions.
  4. Score the designA transparent, interpretable rubric turns the measured metrics into a 0–100 design score — validated separately (below).
The moat: because the design facts are measured deterministically, the review can cite them by number ("body text is 32pt, above the 24pt legibility floor; contrast 16:1") instead of a vision model guessing. A paper reviewer can't do this — papers aren't visual.

Method & evaluation — the honest version

The design score is validated

With no labelled "good poster" dataset to train on, the design score is a transparent rubric, and it's tested the way you test any label-free metric: controlled degradation. Real posters are perturbed in known-bad directions, the metrics are re-measured from the perturbed pixels, and the score must drop. It does, reliably:

DegradationScore dropped
Reduce contrast83%
Clutter the palette83%
Blur the poster100%

What it does not claim: an acceptance oracle

The obvious ambition — predict a reviewer's score from the poster — turns out to be structurally impossible, and PosterReview says so rather than faking a number. We linked 293 real ICLR posters to their OpenReview reviewer ratings and tested two predictors:

Predictor → reviewer ratingCorrelation
Measured design metrics≈ 0.00
LLM-judged content scores−0.04

Both are null, for one reason: posters only exist for accepted papers. A rejected paper never gets a poster, so any poster corpus is ~96% accepted — there is no negative class to learn from, and the leftover poster-vs-spotlight-vs-oral gradation is a band even two human reviewers barely separate (their agreement is 0.41, with the full paper in hand). Stanford's paper-scoring result doesn't transfer, because papers have rejects and posters don't.

So PosterReview reports a design score and a content review as signals for improving a poster — not as a prediction of whether it will be accepted. That's the defensible claim, and it's the one that's actually useful to an author.

Zero cost, on your own compute

Every stage runs locally. The measurement, retrieval, and design score are pure computation. The review runs on an open model via one of two interchangeable engines:

  • MLX — a compact model on Apple Silicon, for laptops.
  • Ollama — a 70B-class model on a workstation/GPU box (e.g. an NVIDIA DGX Spark), reached over an encrypted SSH tunnel, for near-frontier quality.

No API keys, no metered services, no data leaving your machines.

Run it

git clone <this repo> && cd posterreview
uv venv .venv && uv pip install -r research/requirements.txt
.venv/bin/python app/app.py          # web UI at http://127.0.0.1:5000
# or a single poster from the CLI:
.venv/bin/python research/scripts/review.py path/to/poster.pdf

Reviews are generated automatically and may contain errors — use them as guidance and apply your own judgment.
